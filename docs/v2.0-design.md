# DeepSeek Code Agent v2.0 设计书

> 从"能用"到"让人爱不释手"——性能、体验、生态的全面进化

## 总览

v1.x 完成了核心框架 + 桌面端的全链路打通。v2.0 聚焦六个维度，按 P0/P1/P2 分层推进：

| 优先级 | 维度 | 核心目标 |
|--------|------|---------|
| P0 | 性能与资源优化 | 让快更快，让大上下文用得起 |
| P0 | 测试与质量升级 | 自适应测试 + 性能守护 + 安全扫描 |
| P1 | 代码理解深度 | 语义重构 + 运行时调试 + 架构嗅觉 |
| P1 | 桌面端体验 | 多会话 + 语音 + 拖拽 + 主题 |
| P1 | 开放生态 | 工具 SDK + MCP 深度对齐 + A2A |
| P2 | 观测与维护 | OpenTelemetry + 调试面板 + 错误反馈 |

## 一、性能与资源优化（P0）

### 1.1 工具调用并发执行

**现状**：`AgentLoop` 串行处理 `tool_calls`，即使模型一次返回 3 个 `read_file`，也要逐个等待。

**方案**：
- 在 `agent/loop.py` 的 `_execute_tools` 方法中，当 `tool_calls` 列表长度 > 1 时，用 `asyncio.gather` 并发执行
- 需要标注哪些工具可并发（只读工具可并发，写工具需串行）——在 `Tool` 基类加 `read_only: bool` 属性
- 并发写操作需做文件锁检测（复用 `parallel.py` 的 `writes_files` 冲突检测）

**实现文件**：`agent/loop.py`，`tools/base.py`

### 1.2 知识图谱增量更新

**现状**：`ingest_project` 每次全量扫描目录，大项目耗时数十秒。

**方案**：
- 集成 `watchdog` 库，启动文件系统监听
- 文件变动时只重新解析变更文件，更新 `RelationGraph` 中的对应节点和边
- 在 `knowledge/ingest.py` 新增 `IncrementalIngestor` 类
- 提供快照持久化（JSON），启动时加载快照而非全量重扫

**新增依赖**：`watchdog>=3.0`（加入 `knowledge` extras）

**实现文件**：`knowledge/ingest.py`，新增 `knowledge/watcher.py`

### 1.3 动态上下文预算

**现状**：长对话中 token 无限增长，逼近模型上限时简单截断，可能丢失关键信息。

**方案**：
- 实现 `ContextBudget` 策略管理器，为不同类型内容设定优先级和 token 预算：
  - P0（不可裁剪）：当前任务指令、系统 prompt
  - P1（可总结）：工具返回结果 → 保留摘要 + 完整结果存 memory
  - P2（可丢弃）：历史对话轮次 → 超过 N 轮自动总结
- 预算超限时自动触发总结（调用 DeepSeek chat 模型），而非截断
- 在 `AgentLoop` 每轮开始时检查 `total_tokens / budget_ratio`，超过阈值触发裁剪

**实现文件**：新增 `agent/context_budget.py`，修改 `agent/loop.py`

### 1.4 后端冷启动优化

**现状**：桌面端通过 `child_process.spawn('python', ...)` 启动 FastAPI 服务，冷启动需 2-5 秒。

**方案**：
- **短期**：后端常驻模式（桌面端启动时 fork 后台进程，关闭窗口时不断开）
- **中期**：PyInstaller 打包为单文件 exe，启动加速到 <1 秒
- **长期**：用 Rust/Golang 重写热路径（CLI 入口 + 工具注册），Python 只做模型调用

**实现文件**：`server/app.py`，桌面端 `main/index.js`

## 二、代码理解的深度进化（P1）

### 2.1 语义级重构（auto_refactor）

**现状**：`analyze_impact` 能分析影响范围，但重构仍需人工逐个修改。

**方案**：
- 新增 `auto_refactor` 工具，输入重构描述（如"把 `old_func` 重命名为 `new_func`"）
- 利用知识图谱定位所有调用点、注释、文档中的引用
- 生成批量修改预览（Diff 格式），推送至 DiffManager 审批
- 支持的重构类型：
  - 重命名（函数/类/变量）
  - 提取函数
  - 内联函数
  - 移动模块（含 import 更新）

**实现文件**：新增 `tools/refactor.py`

### 2.2 运行时调试（DAP 集成）

**现状**：Agent 只能通过 `run_shell` 执行命令，看到 stdout/stderr，无法实时观察运行时状态。

**方案**：
- 实现 DAP (Debug Adapter Protocol) 客户端，连接 Python 调试器（`debugpy`）
- 新增调试工具集：
  - `debug_start` — 启动调试会话，设置断点
  - `debug_continue` / `debug_step_over` / `debug_step_into` — 执行控制
  - `debug_get_variables` — 查看当前作用域变量
  - `debug_evaluate` — 在断点处执行表达式
- 桌面端 TerminalPanel 新增调试视图（变量面板 + 调用栈）

**新增依赖**：`debugpy>=1.8`（加入 `debug` extras）

**实现文件**：新增 `tools/debug.py`，桌面端新增 `DebugPanel.jsx`

### 2.3 架构嗅觉检测

**现状**：知识图谱有数据但没有"判断力"。

**方案**：
- 在知识图谱上运行架构检测规则：
  - 循环依赖（图中的环）
  - 上帝类（入度/出度超过阈值）
  - 长函数（AST 节点行数 > 50）
  - 过度耦合（扇入/扇出比异常）
- 新增 `arch_check` 工具，返回结构化报告 + 重构建议
- 结果推送至桌面端 DiffViewer，以 annotation 形式标注问题位置

**实现文件**：新增 `tools/arch_check.py`，修改 `knowledge/graph.py`（新增分析方法）

## 三、测试与质量保障升级（P0）

### 3.1 自适应测试用例生成

**现状**：`generate_tests` 生成测试后不管成败。

**方案**：
- 在 `testing.py` 的 `generate_tests` 中增加反馈循环：
  1. 生成测试 → 运行 → 收集失败结果
  2. 将失败结果（异常信息 + 覆盖缺失）反馈给 LLM
  3. LLM 补充边界用例，最多 3 轮
- 跟踪每轮的覆盖率变化，确保单调递增

**实现文件**：修改 `tools/testing.py`

### 3.2 性能基准守护

**现状**：代码修改后无性能回归检测。

**方案**：
- 新增 `benchmark` 工具：
  - 基于 `pytest-benchmark`，用户预设性能基线（JSON 格式）
  - Agent 修改代码后自动运行基准测试
  - 性能劣化超过阈值（默认 5%）标记为 WARNING
  - 结果推送至 DiffManager，与代码变更一起审批
- 在桌面端 ToolCallCard 中展示性能对比柱状图

**新增依赖**：`pytest-benchmark>=4.0`（加入 `testing` extras）

**实现文件**：新增 `tools/benchmark.py`

### 3.3 安全扫描集成

**现状**：Agent 写入文件前无安全检查。

**方案**：
- 在 `edit_file` / `write_file` 执行前增加安全扫描钩子：
  - Python 文件 → `bandit` 扫描
  - JS/TS 文件 → 内置规则检查（eval、innerHTML、SQL 拼接等）
  - 依赖文件 → `npm audit` / `pip audit`
- 扫描结果标注在 Diff 预览界面，高严重度问题阻止写入
- 用户可配置安全策略（忽略规则、严重度阈值）

**新增依赖**：`bandit>=1.7`（加入 `security` extras）

**实现文件**：新增 `tools/security.py`，修改 `tools/fs.py`（添加钩子）

## 四、桌面端体验打磨（P1）

### 4.1 多会话与工作区恢复

**方案**：
- Sidebar 新增会话标签页列表，支持新建/切换/关闭
- 每个会话绑定独立后端 WebSocket 连接
- 应用关闭时序列化所有会话状态到 `~/.deepseek-agent-desktop/sessions/`
- 重新打开时恢复对话历史、未审批 Diff、任务进度

**实现文件**：新增 `SessionManager.js`，修改 `Sidebar.jsx`、`App.jsx`

### 4.2 语音输入

**方案**：
- 集成 OpenAI Whisper API（或本地 `whisper.cpp`）
- ChatPanel 底部新增麦克风按钮
- 录音 → 转写 → 作为用户输入发送
- 支持中英文混合

**新增依赖**：`openai`（前端调用，或后端代理）

**实现文件**：新增 `VoiceInput.jsx`，后端新增 `/api/transcribe` 端点

### 4.3 截图编码

**方案**：
- ChatPanel 支持粘贴截图（ClipboardEvent）
- 截图发送至后端，调用 DeepSeek VL（视觉语言模型）识别内容
- 返回结构化描述（代码、错误信息、UI 布局等）

**实现文件**：修改 `ChatPanel.jsx`，后端新增 `/api/analyze-image` 端点

### 4.4 可定制主题与布局

**方案**：
- 内置 3 款主题：DeepSeek 紫（默认）、Monokai 暗色、Solarized 浅色
- CSS 变量驱动，支持用户自定义覆盖
- 布局预设：默认（左聊天 + 右 Diff）、全屏终端、上下分屏
- 拖拽调整面板大小（react-split-pane）

**新增依赖**：`react-split-pane`（或自实现）

**实现文件**：新增 `themes/` 目录，修改 `App.jsx`、`global.css`

## 五、开放生态（P1）

### 5.1 工具插件 SDK

**现状**：新增工具需修改 `tools/__init__.py` 和 `tools/` 目录。

**方案**：
- 设计极简 Tool SDK：
  ```python
  from deepseek_agent.sdk import tool

  @tool(name="query_feishu", description="查询飞书文档")
  def query_feishu(doc_url: str) -> str:
      """查询飞书文档内容"""
      # 用户实现
      return content
  ```
- 工具注册到 `~/.deepseek-agent/plugins/` 目录
- AgentLoop 启动时自动扫描加载
- 桌面端提供插件管理界面（安装/卸载/启用/禁用）

**实现文件**：新增 `sdk/` 包（`__init__.py`、`loader.py`）

### 5.2 MCP 深度对齐

**现状**：已有 MCP Client（stdio 模式），但功能有限。

**方案**：
- **MCP 服务器增强**：暴露所有 24+ 工具为 MCP 资源，支持 Claude Desktop 等客户端调用
- **MCP 客户端增强**：支持 SSE 传输（除现有 stdio），连接 Puppeteer、Postgres 等外部 MCP 服务器
- **工具桥接**：MCP 服务器提供的工具自动注册到 ToolRegistry，对 Agent 透明可用
- 配置文件：`~/.deepseek-agent/mcp_servers.json`

**实现文件**：修改 `mcp/client.py`、`mcp/server.py`

### 5.3 A2A (Agent-to-Agent) 协议

**方案**：
- 定义 A2A 接口：Agent 发现 → 能力协商 → 任务委派 → 结果回收
- 内置 A2A Server（HTTP + WebSocket）
- 示例：浏览器自动化 Agent、数据库管理 Agent
- 配置文件：`~/.deepseek-agent/agents.json`

**实现文件**：新增 `a2a/` 包（`protocol.py`、`server.py`、`client.py`）

### 5.4 Prompt 与任务模板

**方案**：
- 模板格式：YAML + Markdown，包含 prompt 链 + 工具约束 + 上下文模板
- 模板仓库：`~/.deepseek-agent/templates/` + GitHub 同步
- 桌面端模板浏览器：分类展示，一键加载

**实现文件**：新增 `templates/` 目录，桌面端新增 `TemplateBrowser.jsx`

## 六、观测与维护（P2）

### 6.1 OpenTelemetry 集成

**方案**：
- 每次 Agent 循环生成一个 Trace，每个工具调用生成一个 Span
- 本地导出（`~/.deepseek-agent/traces/`），用户可控
- 桌面端开发者面板：可视化思考链路、耗时、token 消耗

**新增依赖**：`opentelemetry-api>=1.20`，`opentelemetry-sdk>=1.20`

**实现文件**：新增 `core/telemetry.py`，修改 `agent/loop.py`、`tools/base.py`

### 6.2 错误报告与反馈

**方案**：
- Agent 连续 3 次工具调用失败时，自动生成错误摘要
- 桌面端一键"报告此会话"按钮
- 收集：日志 + 上下文摘要 + Agent 思考链 + 用户描述
- 发送至用户自建的反馈服务器（本地优先，不上传第三方）

**实现文件**：新增 `core/feedback.py`，桌面端新增 `FeedbackDialog.jsx`

## 与 v1.x 的兼容性

- 所有 v1.x 工具和 API 保持向后兼容
- 新工具通过 `ToolRegistry.register()` 加入，不修改现有工具代码
- 桌面端新功能通过 Feature Flag 控制（`~/.deepseek-agent-desktop/config.json`）
- MCP/A2A 协议为纯增量，不影响现有 stdio 模式
