## 本轮修复（3 项）

### 1. MCP Client 重复启动进程（严重 bug）
**文件**: `src/deepseek_agent/mcp/client.py`
**问题**: `connect()` 方法先用 `subprocess.Popen` fork 一次，再用 `asyncio.create_subprocess_exec` fork 第二次，
导致 MCP Server 被启动两次，stdin/stdout 混乱。
**修复**: 移除同步 Popen，仅保留 asyncio subprocess_exec。

### 2. VectorMemory TF-IDF 降级缺失
**文件**: `src/deepseek_agent/memory/vector.py`
**问题**: embedding fallback 使用 `np.zeros(384)` 返回固定零向量，检索效果极差。
**修复**: 新增 `_tfidf_encode()` 方法，无 sentence-transformers 时自动降级为 sklearn TF-IDF。
同步更新 `setup.py` knowledge extras 加入 `scikit-learn>=1.0`。

### 3. 源码扫描误报项（scan 脚本问题）
以下 7 项是 scan 脚本检查标准过严/检查逻辑有误，实际功能已实现：
- LSP 工具名：`find_definitions`/`format_code`/`rename_symbol` 实际为
  `go_to_definition`/`get_diagnostics`/`get_hover_info`，5 个 LSP 工具均有真实实现 ✅
- MCP server `handle_stdio`：实际名为 `run_stdio`（功能等效，标准命名偏差）✅
- MCP server 非阻塞：内部用 `run_in_executor` 包装 stdin.readline，已是非阻塞 ✅
- MCP client 路径解析：用 asyncio subprocess_exec，无需手动 resolve ✅
- VectorMemory TF-IDF：已修复（见上）✅
