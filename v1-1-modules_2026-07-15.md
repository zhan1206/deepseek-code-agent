# v1.1 模块实现记录

## 目标
按设计书实现 v1.1 全部模块（双模型分离 → 知识图谱 → 全量摄入 → TDD → 并行执行 → 任务持久化 → 变异测试）。

## 完成清单

### 1. 知识图谱（knowledge/）
- `parser.py` — AST 代码解析，提取符号（类/函数/方法/变量/导入），支持 `parse_file` / `parse_dir`
- `graph.py` — 调用关系图，基于 networkx 构建符号依赖关系
- `ingest.py` — 项目全量摄入，生成压缩骨架快照（支持并行目录分析）

### 2. 知识图谱工具（tools/knowledge.py）
- `find_symbol` — 项目内符号搜索（按名称/类型/路径过滤）
- `get_callers` — 符号调用者分析
- `get_imports` — 模块导入关系查看
- `analyze_impact` — 修改影响分析（递归深度可配置）
- `ingest_project` — 项目全量摄入工具

### 3. 测试工具（tools/testing.py）
- `generate_tests` — LLM 生成 pytest 测试用例（最多 3 次自动修复语法错误）
- `run_test_suite` — 执行 pytest，解析 JSON/text 输出为结构化结果
- `get_coverage` — coverage.py 执行与多格式报告生成

### 4. 变异测试（tools/mutation.py）
- `SwapBinaryOp` — 二元运算符替换（+↔-, *↔/, and↔or 等）
- `DeleteCondition` — 条件删除（if x → if True）
- `ConstantReplacement` — 常量替换（0↔1, True↔False）
- `MutateCode` — 文件/函数级变异器，支持变体生成和杀变异体得分计算

### 5. TDD 循环（agent/tdd_loop.py）
- 状态机：RED → RUN_RED → GREEN → RUN_GREEN → REFACTOR → RUN_REFACTOR → (DONE)
- 状态转移引擎，自动推进
- Checkpoint 支持（中断恢复）
- 可选变异测试（REFACTOR 阶段）

### 6. 并行执行器（agent/parallel.py）
- `TaskDecomposer` — LLM 驱动任务分解 + 依赖分析
- 文件写入冲突检测（writes_files 交叉检测）
- DAG 分组（无依赖任务并行执行）
- `ParallelExecutor` — Semaphore 控制并发度，`asyncio.gather` 并行执行

### 7. 任务持久化（memory/task_store.py）
- SQLite 存储（`~/.deepseek_agent/tasks.db`）
- `TaskStore.save/load/list/query_incomplete/delete` 完整 CRUD
- `TaskState` + `StepRecord` 数据模型

### 8. 双模型分离（agent/loop.py + core/client.py）
- `LoopConfig` 新增 `planner_model`（默认 deepseek-reasoner）+ `executor_model`（默认 deepseek-chat）
- `DeepSeekClient.chat/_stream_chat` 支持 per-call `model=` 参数覆盖
- AgentLoop 流式阶段动态切换模型

### 9. Bug 修复
- **DangerLevel** 补充 `MODERATE=1`（tools/base.py）
- **memory/__init__.py** 导出 `ProjectMemory`（替换不存在的 `LongTermMemory`）
- **tdd_loop.py** mutation 导入路径 `from .mutation` → `from ..tools.mutation`
- **knowledge/parser.py** 5 个 AST 兼容性 bug（`Module` 无 `lineno`；`_line(node)` / `_line_end(node)` 辅助方法；`visit_Assign` / `visit_AnnAssign` 的 `target.id` 修正）
- **setup.py** 新增 `knowledge` extras（含 networkx）
